# 验证集方法 (The Validation Set Approach)

## 核心原理
这是最简单、最直观的重采样方法。
我们将现有的数据集随机划分为两部分：
1.  **训练集 (Training set)**：用于训练模型。
    * 通常占比 50%-80%。
2.  **验证集 (Validation/Hold-out set)**：用于评估模型性能。
    * 模型在训练时**绝对不能**看到这部分数据。

$$n_{total} = n_{train} + n_{val}$$

## 流程
1.  随机分割数据。
2.  在训练集上拟合模型 $\hat{f}$。
3.  将模型代入验证集，计算均方误差 (MSE) 或错误率。

## 优缺点分析 (编程手必看)

| 维度 | 特点 | 评价 |
| :--- | :--- | :--- |
| **实现难度** | 极低 | 直接用 `sklearn.model_selection.train_test_split` 即可。 |
| **计算成本** | 低 | 只需要跑一次模型。 |
| **稳定性** | **差 (High Variance)** | 这是一个致命伤。**分割点的随机性**会导致评估结果剧烈波动（见 ISLP 图 5.2）。运气好分得好，误差就低；运气不好，误差就高。 |
| **偏差 (Bias)** | **高** | 因为我们只用了一部分数据（如 50%）来训练。根据学习曲线，数据越少模型表现越差。这导致我们**高估**了测试误差（把模型想得比实际更菜）。 |
